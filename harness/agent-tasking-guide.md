# 에이전트 사용 가이드

이 하네스는 사용자가 복잡한 에이전트 이름을 몰라도 쓸 수 있게 구성되어 있습니다. 아래 명령 중 하나만 Codex에게 말하면 됩니다.

자세한 내부 라우팅 규칙은 `harness/simple-commands.md`에 있습니다.

## 가장 자주 쓰는 명령

### 1. 오늘 분석해줘

```text
오늘 분석해줘
```

하는 일:

- Alpaca watchlist를 기준으로 종목을 봅니다.
- Alpaca MCP 데이터와 현재 웹 리서치를 함께 사용합니다.
- 종목별 일별/주별/월별 트렌드를 봅니다.
- 추천 후보와 리스크를 정리합니다.
- 결과를 `wiki/`에 저장합니다.
- 주문은 제출하지 않습니다.

### 2. 관심종목 분석해줘

```text
관심종목 분석해줘
```

하는 일:

- Alpaca watchlist에 있는 종목만 리서치합니다.
- 각 종목의 thesis, 촉매, 리스크, 트렌드를 업데이트합니다.
- 원천 자료를 `wiki/raw/sources/`에 남깁니다.
- 주문은 제출하지 않습니다.

### 3. 특정 종목 분석해줘

```text
AAPL 분석해줘
```

여러 종목도 가능합니다:

```text
AAPL MSFT NVDA 분석해줘
```

하는 일:

- 지정한 티커만 리서치합니다.
- 종목끼리 비교하고 순위, 신뢰도, 핵심 리스크를 정리합니다.
- 관련 종목 페이지를 `wiki/tickers/`에 업데이트합니다.
- 주문은 제출하지 않습니다.

### 4. 포트폴리오 점검해줘

```text
포트폴리오 점검해줘
```

하는 일:

- Alpaca paper 계좌 상태를 확인합니다.
- 현금, buying power, 포지션, 미체결 주문을 정리합니다.
- `wiki/portfolio/current.md`를 업데이트합니다.
- 주문은 제출하지 않습니다.

### 5. 리밸런싱 계획 짜줘

```text
리밸런싱 계획 짜줘
```

하는 일:

- 현재 계좌와 포지션을 기준으로 배분안을 만듭니다.
- 최대 80% 투자, 최소 20% 현금, 종목당 최대 20% 규칙을 지킵니다.
- 주문 계획 JSON을 만들고 `scripts/check-risk-policy.py`로 검증합니다.
- 주문은 제출하지 않습니다.

### 6. paper 주문까지 실행해줘

```text
paper 주문까지 실행해줘
```

하는 일:

- 최신 리밸런싱 계획을 확인합니다.
- 필요한 경우 새 리밸런싱 계획을 만듭니다.
- paper mode, 시장 개장, fresh quote, risk gate를 모두 확인합니다.
- 통과할 때만 Alpaca MCP로 paper 주문을 제출합니다.
- 주문 후 바로 거래 후 점검을 실행합니다.

### 7. 거래 후 점검해줘

```text
거래 후 점검해줘
```

하는 일:

- 최근 주문, 체결, 취소, 미체결 주문을 확인합니다.
- 포지션, 현금, buying power를 다시 대조합니다.
- portfolio wiki와 관련 리포트를 업데이트합니다.
- 새 주문은 제출하지 않습니다.

### 8. 거래 회고해줘

```text
거래 회고해줘
```

하는 일:

- 체결된 paper 거래와 현재/청산 포지션을 검토합니다.
- 매수/매도 당시의 종목 분석, 일일 리포트, 주문 계획, raw source를 다시 읽습니다.
- 실제 결과와 비교해 어떤 판단이 좋았고 어떤 판단이 부족했는지 기록합니다.
- 종목별 회고를 `wiki/reviews/trades/`에 남깁니다.
- 반복적으로 확인된 교훈은 `wiki/policies/recommendation-policy.md`에 반영합니다.
- 새 주문은 제출하지 않습니다.

### 9. 위키 정리해줘

```text
위키 정리해줘
```

하는 일:

- 오래된 주장, 누락된 출처, 고아 페이지, index 누락을 찾습니다.
- 모순된 판단을 표시합니다.
- 정리 리포트를 `wiki/analyses/`에 저장합니다.
- 주문은 절대 하지 않습니다.

## 안전한 기본 동작

- 명령에 `주문`, `매수`, `매도`, `실행`이 없으면 주문하지 않습니다.
- 모든 주문은 paper trading만 허용됩니다.
- 미국 주식/ETF만 허용됩니다.
- whole-share day limit order만 허용됩니다.
- 주문 전에는 반드시 `scripts/check-risk-policy.py`를 통과해야 합니다.
- API key나 `.env` 내용은 출력하지 않습니다.

## 추천 사용 흐름

처음에는 이렇게 쓰면 충분합니다:

```text
오늘 분석해줘
```

결과가 괜찮으면:

```text
리밸런싱 계획 짜줘
```

계획까지 괜찮으면:

```text
paper 주문까지 실행해줘
```

주문 뒤에는:

```text
거래 후 점검해줘
```

거래 결과가 쌓이면:

```text
거래 회고해줘
```

주기적으로:

```text
위키 정리해줘
```

## 조금 더 구체적으로 말하고 싶을 때

짧은 명령 뒤에 조건만 덧붙이면 됩니다.

```text
오늘 분석해줘. 반도체 종목을 특히 자세히 봐줘.
```

```text
AAPL MSFT NVDA 분석해줘. 이번 주 뉴스와 실적 관련 리스크를 중심으로 봐줘.
```

```text
리밸런싱 계획 짜줘. 신규 주문은 최대 3개까지만 제안해줘.
```

```text
포트폴리오 점검해줘. 현재 보유 종목별 비중도 같이 보여줘.
```

사용자는 짧게 말하고, 하네스는 내부적으로 필요한 에이전트와 워크플로우를 선택합니다.
